Output record for `kwin_dbus_callback_test.py`.

**Result: SUCCESS.**  The D-Bus callback round-trip from a KWin JS script to a

### kwin_geometry_screen_test.md

[kwin_geometry_screen_test.md](kwin_geometry_screen_test.md)

Output record for `kwin_geometry_screen_test.py`.

**Result: MIXED.**  All three geometry-setter approaches for changing window
*size* silently failed — this is expected Wayland behaviour (compositor-
initiated resize is ignored by clients).  Direct `w.output = screenObject`
assignment threw a `TypeError: Cannot assign to read-only property`.
Geometry-based screen assignment (moving the window's centre-point into the
target screen's bounding rectangle) **worked** and was adopted as the library
implementation.  See ADR-260317-06.

### kwin_introspect.md

[kwin_introspect.md](kwin_introspect.md)

Output record for `kwin_introspect.py`.

**Result: INFORMATIONAL.**  Walked the full `org.kde.KWin` D-Bus object tree.
Key finding: `getWindowList()` is absent from this KWin version.  The only
window-retrieval methods at `/KWin` are `queryWindowInfo()` (interactive click)
and `getWindowInfo(uuid)` (requires a UUID already known to the caller).
This confirmed that `workspace.windowList()` inside the KWin JS scripting
sandbox is the only practical way to enumerate windows.  See ADR-260317-04.

### kwin_mutation_test.md

[kwin_mutation_test.md](kwin_mutation_test.md)

Output record for `kwin_mutation_test.py`.

**Result: INFORMATIONAL / MIXED.**  Confirmed that `workspace.desktops` and
`workspace.screens` return full enumeration data (UUID, name, number, geometry).
`internalId` lookup via `workspace.windowList()` is reliable.  `w.setMaximize()`

### kwin_property_test.md

[kwin_property_test.md](kwin_property_test.md)

Output record for `kwin_property_test.py`.

**Result: FAILURE (definitive).**  Run on 2026-03-21 against a live
`KWin::XdgToplevelWindow` object (Thunderbird).  All three tests failed:
`TypeError: Property 'setProperty' of object KWin::XdgToplevelWindow(...) is
not a function`.  The window was found correctly — the failure is specific to
the `setProperty` / `property` API, which KWin's JS sandbox deliberately does
not expose.

This result was definitive: Qt dynamic properties cannot be used for window
tracking across script loads.  The consequence is that `kwin_realign.py` must
use session-state file tracking (written by `kwin_restore.py` at restore time)
rather than in-process UUID tagging.  See ADR-260321-01.  The test was run a
second time with a different window UUID to rule out a lookup error — the
failure mode was identical.

### kwin_tile_test.md

[kwin_tile_test.md](kwin_tile_test.md)

Output record for `kwin_tile_test.py`.

**Result: SUCCESS.**  All eight `slotWindowQuickTile*` methods were confirmed
present and callable.  The slot methods operate on `workspace.activeWindow` —
the target window must be set as active before calling them.  Tile-left was
confirmed to halve the window width (verified from the "before" geometry in the
subsequent tile-right test, which showed the halved width).  Geometry read-back
in the same script may be stale due to KWin commit timing; this does not affect
production use.  Key caution recorded: `slotWindowQuickTile*` puts the window
into a tile mode that blocks subsequent `setMaximize()` calls — never tile and
then immediately maximize.

### eis_test.py

[eis_test.py](eis_test.py)

**Purpose:** Diagnostic for the KWin private EIS (`org.kde.KWin.EIS.RemoteDesktop`)
input injection interface.  Connects via D-Bus, initialises a `libei` sender
context using `ctypes`, runs a 5-second event loop, and reports which input
device capabilities became available.

**Result: FAILED (design limitation).**  KWin's private EIS interface grants
`POINTER_ABSOLUTE`, `TOUCH`, `BUTTON`, and `SCROLL` capabilities but
deliberately withholds `KEYBOARD`.  No amount of requesting `KEYBOARD`
capability produces a keyboard device.  This led to the decision to use the
XDG RemoteDesktop portal instead.  Additional bugs uncovered during testing:
`pydbus` returns the fd list index (not the real fd), causing the initial
connection to land on stdin (`fd=0`); a segfault from untyped `argtypes` in
`ei_seat_bind_capabilities` was also fixed.  See `INVESTIGATION_LOG.md` for
the full sequence and ADR-260323-01 for the architecture decision.

### kwin_click.py

[kwin_click.py](kwin_click.py)

**Purpose:** Visual test for the coordinate math used by `{WCLICK}` tokens in
`kwin_restore.py` input action injection.  Opens a 1000×1000 window with a
dot grid every 25 px.  On each mouse click, draws a 48 px red filled circle
with a white crosshair at the click position and displays both
window-relative and absolute compositor coordinates in the title bar.

**Result: WORKING.**  Confirmed that `Qt.mapToGlobal()` returns the same
absolute compositor coordinate that the portal's `NotifyPointerMotionAbsolute`
expects, validating the `{WCLICK x, y}` coordinate math.  The dot appears
exactly where the user clicked.  Note: the ideal follow-on test — using
`kwin_restore.py`'s portal injection to send `{WCLICK}` tokens to this window
and comparing the dot positions — was not completed, as the full-screen
`ClickIndicator` overlay intercepts input before it reaches the canvas.
This remains a future improvement.

### kwin_secret_service.py

[kwin_secret_service.py](kwin_secret_service.py)

**Purpose:** Diagnostic and test tool for the `org.freedesktop.secrets` Secret
Service protocol.  Supports three commands: `introspect` (print raw D-Bus
introspection XML), `list [--verbose]` (enumerate all items in the default
collection without fetching secrets), and `lookup <key>` (fetch and print the
secret for a named entry).  Uses `Gio.DBusProxy` throughout for correct
interface negotiation, making it compatible with any compliant Secret Service
implementation (KeePassXC, GNOME Keyring, KWallet bridge, etc.).

**Result: WORKING.**  Written to diagnose an `UnknownInterface` error in the
initial `secret_service.py` library implementation.  The `introspect` command
revealed that `Gio.DBusProxy` (not raw `Gio.DBusConnection.call_sync()`) was
required for correct interface negotiation.  The library was subsequently
rewritten to use `DBusProxy` throughout, matching this script's approach.
The `list` and `lookup` commands were confirmed working against a live
KeePassXC instance with Secret Service Integration enabled.

### portal_keyboard_test.py

[portal_keyboard_test.py](portal_keyboard_test.py)

**Purpose:** Test the full XDG RemoteDesktop portal keyboard injection stack.
Implements the complete portal handshake — `CreateSession`, `SelectDevices`,
`Start` (including the KDE authorization dialog), and `NotifyKeyboardKeycode`
— and sends a single `t` keystroke to the focused window.  Saves a restore
token to skip the authorization dialog on subsequent runs.

**Result: SUCCESS.**  After fixing three bugs (request path construction,
synchronous treatment of async calls, and wrong `Gio.signal_subscribe` callback
arity), the full handshake completed, the KDE authorization dialog appeared and
was approved, and the `t` keystroke was delivered to the focused terminal.
This confirmed the complete XDG portal keyboard injection stack and drove
ADR-260323-01 (adopt XDG RemoteDesktop portal as the input injection mechanism).
See `INVESTIGATION_LOG.md` stages 5 and 6 for the full detail.
